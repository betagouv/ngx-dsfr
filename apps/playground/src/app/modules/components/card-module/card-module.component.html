The DsfrCardComponent purpose is to represent card element.

It is based on the Card open_in_new component in the DSFR open_in_new .

It has several @Input() and @Output() properties:

The label is the title, it's displayed in an H3 tag. The title of the card link. Image to add a design element to the card. It's displayed in the image alt attribute (accessibility). It must be an icon class, begin with 'fr-icon' It can be used to display file information when download input is true. It must be an icon class, begin with 'fr-icon' Whether the badge is small or not It is used only if align is set to horizontal. By default this value is undefined, its ration is 40%. It displayed the download design card It displayed a footer above the card. The content of the footer is displayed by ng-content. You can refer to DSFR Documentation to see how to add some content (buttons, links, ...) It display a list of DsfrBadge on the top of the card. It display a list of DsfrBadge on the content of the card. Badges can't be displayed with tags. It display a list of DsfrTag on the content of the card. Tags can't be displayed with badges. It emits on click on the card.
