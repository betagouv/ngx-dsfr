DsfrCheckboxComponent

The DsfrCheckboxComponent purpose is to represent checkboxes, either just one, or a group of several of them.

It is based on the Case Ã  cocher open_in_new component in the DSFR open_in_new .

It has several @Input() properties:

This property represents the text displayed on top of a whole group of checkboxes and enables the end-user to know how the checkboxes in this group are related and what they are about. The ID used for the legend HTML tag . An optional text to help end-users understand the purpose of a group of checkboxes. It is a way to give more details and complement the legend. Only displayed for a group of several checkboxes. The value for the name attribute of the checkboxes. The value must be an array of objects satisfying the CheckboxItem interface : each object must have an id property, a label property and a value property. It may also have an optional hint property. All these properties have string values. Whether the checkboxes in a group will be displayed on top of each other ( when it is false ) or one after the other, on the same row. Whether a whole group of checkboxes is disabled or not. Whether the checkboxes are displayed in a small size or one more fitted for a mobile usage where you need a large enough touch area. Whether the control ( represented by either a single checkbox or a group of them ) is in error mode or not. The error message to be displayed when in error mode. Whether the control ( represented by either a single checkbox or a group of them ) is in success mode or not. The success message to be displayed when in success mode.
