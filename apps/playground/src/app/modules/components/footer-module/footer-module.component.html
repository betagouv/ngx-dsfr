DsfrFooterComponent

The DsfrFooterComponent purpose is to represent the footer of an application.

It is based on the Footer open_in_new component in the DSFR open_in_new .

It has several @Input() properties:

The part of the French government ultimately in charge of the web application. It must respect some rules on how to break lines open_in_new . The path on your server to the logo of the operator, the agency within the institution, in charge of the web application. The alternative text for the logo of the operator. Necessary for accessibility: it MUST have at least the text visible on the logo illustration. The name of your web application. A short description of what your application does or useful information. The path, corresponding to the Angular Route that you want the user to be redirected to when they click on the institution, operator's logo or your app's name. The value must be an array of objects satisfying the FooterCategoryLinks interface : each object must have a title property and a children property, which is an array of objects satisfying the FooterLink interface with a label property, a href property, and an optional isExternal property. The value must be an array of objects satisfying the FooterLink interface: each object must have a label property, a href property, and an optional isExternal property. The value must be an array of objects satisfying the FooterBrand interface: each object must have an image property (containing the path), a href property, and an alt property.
The first element of the array is the main brand, it will be displayed on the left, the others will be displayed on the right. The value must be an array of objects satisfying the FooterLink interface: each object must have a label property, a href property, and an optional isExternal property.
