DsfrLinkComponent

The DsfrLinkComponent purpose is to represent a link to a specific URL, either within or outside the application.

It is based on the Link open_in_new component in the DSFR open_in_new .

It has several @Input() properties

The text actually displayed within the link. Whether the link is supposed to be displayed within a text or not It is added to the href attribute, it can be internal or external. When the link is external (begins by http:// or https://) the target attribute is set to _blank You can import the ElementSize enum, from the @betagouv/ngx-dsfr library entry-point, in your app, to safely provide a value for this property. This input permits to set the component as a link which when the user click on it, it scrolls to the top of the current page. It will display an icon, alone if no label provided, or at the left or right according to the iconAlignment Input.
Here open_in_new is the list of all the icons available. It will align the provided icon at the left or right of the label. This input is not necessary if no label is provided. You can import the ElementAlignment enum, from the @betagouv/ngx-dsfr library entry-point, in your app, to safely provide a value for this property. For accessibility, it will display the title on hover of the link. It is used to permits to the end-users to download a file. The property link value must be the path of the corresponding file. An optional text to help end-users understand the purpose of a link. It also can be used to display the file information when it is a download link.
