DsfrToggleComponent

The DsfrToggleComponent purpose is to represent a toggle element.

It is based on the Toggle open_in_new component in the DSFR open_in_new .

It has several @Input() properties and one @Output() property:

It is required and given to the id attribute of the underlying input tag. It is also used to generate the aria-describedby attribute The label of the toggle. It is mandatory for accessibility reasons. You can, however, choose not to display it if it suits your use-case. Keep in mind though that it will still be present in the DOM so screen-readers can access it. Also, it is recommended not to change the label according to the state of the toggle Whether or not you wish to display the label and the hint, if provided A string explaining what the toggle represents when it's checked A string explaining what the toggle represents when it's not checked It will display a description below the toggle. It is optional and can be hidden when hideTexts is true. You can use this as a way to make the purpose of the toggle more explicit than just with the label Whether the label is on the left or right of the toggle The value for the disabled attribute, for the underlying input tag Whether or not the element will display like a toggle ( choose to activate something or not ) or like a switch ( choose between 2 equally legitimate possibilities ). When set to true, the toggle will not take any color when checked ( or in its activated state ) and the only visual clue that it has changed will be that its round part will have slid to the right Emits the value of the toggle as boolean when the user toggles it
